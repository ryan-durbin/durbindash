## Codebase Patterns
- Project uses CommonJS (require/module.exports), NOT ESM — despite node-fetch v3 being ESM-only, existing code uses require('node-fetch') which fails in Jest. Use Node built-in https module for new server modules to avoid Jest ESM compatibility issues.
- Tests use Jest (jest.spyOn, jest.mock), run with `npx jest`
- Server modules live in server/ directory, exported as { functionName }
- In-memory cache pattern: `const cache = {}; cache[key] = { data, fetchedAt: Date.now() }`
- Export `_cache` reference for test cleanup in beforeEach
